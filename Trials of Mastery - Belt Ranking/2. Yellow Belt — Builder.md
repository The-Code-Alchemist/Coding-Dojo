## Building your first backend application

---

## 🧠 Overview

Welcome to the Yellow Belt.

At this stage, you transition from basic survival skills to actually **building software**.

You will learn how to create a structured Java project, understand build tools, and expose functionality through a simple API.

---

## 🎯 Objectives

After completing the Yellow Belt, you will be able to:

* Create a structured Java project using Maven
* Understand project dependencies
* Build and run a backend application
* Create REST endpoints
* Use IntelliJ effectively for development
* Run your application using a standardized workflow

---

## 🧰 Required Setup

Before starting, ensure you have completed the **White Belt — Survival & Setup**.

You must have:

* Java (LTS, e.g. 21)
* IntelliJ IDEA
* Git
* GitHub account
* Git Bash (for consistent terminal usage)

---

## 📚 Topics Covered

* Introduction to Maven
* Project structure (`src/main/java`)
* Dependencies
* Build lifecycle
* Introduction to Spring Boot
* REST APIs
* Standardized run workflows

---

## 🧪 Assignments

### Assignment 1 — Create a Maven Project

* Create a Maven project
* Understand `pom.xml`
* Run:

```bash
mvn clean install
```

---

### Assignment 2 — Run Your Application

Run your application using Maven:

```bash
mvn spring-boot:run
```

Then run it using the standardized script:

```bash
bash scripts/run-dojo.sh
```

Verify that your application starts correctly.

---

### Assignment 3 — REST Endpoint

Create a simple endpoint:

GET /hello

Response:

```
Hello from Coding Dojo
```

---

### Assignment 4 — Dependency Awareness

* Add a dependency
* Explain what it does
* Verify it works

---

### Assignment 5 — Project Documentation

Update your README:

* What does your app do?
* How to run it?
* What dependencies are used?

## 🚀 Standardized Run Workflow

To ensure a consistent development experience, the Coding Dojo uses a standardized run script.

### Run the application

```bash
bash scripts/run-dojo.sh
```

### Why this matters

* Reduces setup complexity
* Prevents environment inconsistencies
* Aligns with real-world DevOps practices

> “One command. One workflow. No confusion.”

## 🧪 Testing Your Endpoint

Once your application is running, verify your endpoint using different tools.

---

### 🌐 Browser

http://localhost:8080/hello

---

### 🐧 Git Bash / Linux / macOS

```bash
curl http://localhost:8080/hello
```

---

### 🪟 PowerShell (Windows)

```powershell
Invoke-WebRequest http://localhost:8080/hello
```

or:

```powershell
Invoke-RestMethod http://localhost:8080/hello
```

## 🛠️ Git Bash in IntelliJ

To enable Git Bash inside IntelliJ:

1. Go to:
   File → Settings → Tools → Terminal

2. Set Shell path to:

```
C:\dev\Git\bin\bash.exe
```

3. Apply and restart the terminal
