## 🧠 Dependency Rules

In Maven, not all dependencies are treated the same.

### 🔑 Golden Rule

* Managed dependency → **no version**
* Unmanaged dependency → **always specify version**

---

### 🧠 Common Mistake

Removing a version from an unmanaged dependency will NOT fix vulnerabilities.

Always upgrade — never ignore.

### 🔍 How to Verify Dependencies

Always verify your dependency graph:

```bash
mvn dependency:tree
```

Ask yourself:

* Where does this dependency come from?
* Is it direct or transitive?
* Which version is actually used?

> “Never assume your dependencies — inspect them.”

### 📦 Managed Dependencies

These are controlled by Spring Boot (via the parent POM).

Example:

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

👉 Version is handled automatically.

---

### ⚙️ Unmanaged Dependencies

These are NOT controlled by Spring Boot.

Example:

```xml
<dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-lang3</artifactId>
    <version>3.20.0</version>
</dependency>
```

👉 You are responsible for:

* version selection
* upgrades
* security

---

### ⚠️ Important Insight

> “Not all dependencies are equal — some are managed for you, others are your responsibility.”

> “Working code is not enough — controlled dependencies create stable systems.”

## 🧪 Assignments

### Assignment 1 — Dependency Exploration

* List all dependencies in your project
* Explain what each one does
* Identify direct vs transitive dependencies

---

### Assignment 2 — Upgrade a Dependency

* Choose one dependency
* Upgrade it to a newer version
* Verify the dependency change using `mvn dependency:tree`
* Fix any issues that arise

---

### Assignment 3 — Introduce a Breaking Change

* Upgrade to a version that causes an error
* Debug and resolve the issue

---

### Assignment 4 — Toolchain Awareness

Document your development stack:

* Java version
* Maven version
* IntelliJ version
* OS version

Explain why version consistency matters

---

### Assignment 5 — Reproducible Build

* Clone your project on another machine (or simulate)
* Ensure it builds successfully

---

## ⚔️ Trial of Mastery — Orange Belt

To earn your Orange Belt, you must:

* Successfully upgrade at least one dependency
* Resolve a breaking change
* Demonstrate understanding of your toolchain
* Ensure your project builds reproducibly
* Document your findings
