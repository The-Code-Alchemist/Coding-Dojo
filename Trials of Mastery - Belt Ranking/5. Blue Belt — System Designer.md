## 🧠 Core Concept

### 🧱 Single Source of Truth

A model should exist in **one place only**.

Example:

```plaintext
dojo-domain → contains Tutorial
dojo-project → uses Tutorial
```

> “Duplication is the enemy of maintainability.”

---

### 🔗 Module Responsibility

Each module has a clear responsibility:

```plaintext
dojo-domain   → data & models
dojo-project  → application logic (Spring Boot)
```

---

### ⚠️ Multi-module Build Rule

Always build from the root:

```bash
mvn clean install
```

> “A system is built as a whole — not in isolation.”

---

## 🧪 Assignments

---

### Assignment 1 — Create a Domain Module

* Create a new module: `dojo-domain`
* Move your model (`Tutorial`) into this module
* Ensure it contains only pure Java (no Spring)

---

### Assignment 2 — Connect Modules

* Add `dojo-domain` as a dependency in `dojo-project`
* Verify imports work correctly

---

### Assignment 3 — Remove Duplication

* Ensure the model exists in only ONE place
* Remove any duplicate classes

---

### Assignment 4 — Multi-module Build

Run:

```bash
mvn clean install
```

Verify:

* Both modules build successfully
* Dependencies resolve correctly

---

### Assignment 5 — System Understanding

Explain:

* Why the domain module exists
* How modules depend on each other
* What would break if the structure is wrong
