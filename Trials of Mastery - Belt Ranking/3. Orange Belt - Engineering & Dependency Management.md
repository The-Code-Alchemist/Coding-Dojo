## 🧠 Overview

Welcome to the Orange Belt.

At this stage, you transition from building applications to understanding how software is **maintained, evolved, and kept up-to-date**.

You will learn how to manage dependencies, control versions, and maintain a healthy development environment.

---

## 🎯 Objectives

After completing the Orange Belt, you will be able to:

* Understand and manage project dependencies
* Keep your software stack up-to-date
* Handle version changes and breaking updates
* Use Maven beyond basic usage
* Maintain a stable and reproducible development environment

---

## 🧰 Required Setup

You must have completed:

* White Belt
* Yellow Belt
* Green Belt

## 📚 Topics Covered

* Maven deep dive (`pom.xml`)
* Dependency scopes
* Versioning (semantic versioning)
* Updating dependencies
* Breaking changes
* Reproducible builds
* Toolchain awareness

---

## 🧪 Assignments

### Assignment 1 — Dependency Exploration

* List all dependencies in your project
* Explain what each one does
* Identify direct vs transitive dependencies

---

### Assignment 2 — Upgrade a Dependency

* Choose one dependency
* Upgrade it to a newer version
* Fix any issues that arise

---

### Assignment 3 — Introduce a Breaking Change

* Upgrade to a version that causes an error
* Debug and resolve the issue

---

### Assignment 4 — Toolchain Awareness

Document your development stack:

* Java version
* Maven version
* IntelliJ version
* OS version

Explain why version consistency matters

---

### Assignment 5 — Reproducible Build

* Clone your project on another machine (or simulate)
* Ensure it builds successfully

---

## ⚔️ Trial of Mastery — Orange Belt

To earn your Orange Belt, you must:

* Successfully upgrade at least one dependency
* Resolve a breaking change
* Demonstrate understanding of your toolchain
* Ensure your project builds reproducibly
* Document your findings
